Navigation menu techniques

This single file demonstrates how block, inline, inline-block, float, and clear affect navigation items. Use the code as a starter for your site.

1) Block-level links — stacked vertical menu
Home Products About Contact

Each <a> is display:block, so it occupies the full width and stacks vertically.

2) Inline links — text-like flow
Home Services Pricing Blog

Using display:inline keeps the links in the text flow. Useful for small inline controls.

3) Inline-block — horizontal with block-like sizing
Home Shop Features Contact

inline-block allows margins/paddings/widths while remaining inline. No float required. Beware of small gaps caused by whitespace in HTML — two fixes: remove space between tags or set font-size:0 on the parent.

4) Float-based horizontal nav + clearfix
Home Catalog Support Login

Here each link is float:left. The parent uses .clearfix (a pseudo-element that clears floats). Another common technique is to give the parent overflow:auto or overflow:hidden to create a new block formatting context.

This box uses clear:both so it will always sit below the floated nav items.
5) Logo left + float menu right (classic header)
MyLogo
Home Products Contact

Because the header doesn't use a clearfix, we've used overflow:auto on the header to contain its floated children and ensure the header expands to the full height of its contents.

Quick cheat-sheet
  • block — element takes full width; good for vertical menus
  • inline — flows with text; can't set width/height safely
  • inline-block — inline but accepts box properties (padding/width)
  • float — takes element out of normal flow and floats it to a side; must be cleared/contained
  • clear — applied to an element to move it past floated siblings (e.g., clear:both)
